# Capstone Pilot Plan

This plan records how to gather real calibration evidence before a stable curriculum release. It is intentionally a plan; an empty calibration table must not be presented as completed pilot evidence.

## Pilot Cohort

Recruit at least three learners with different contexts:

1. one delivery practitioner without sustained incident ownership;
2. one production operator or SRE;
3. one platform, application, or data specialist using an overlay.

Do not collect credentials, customer data, or private employer material. Learners may use the local reference system and design-only alternatives.

## Pilot Procedure

1. Record the learner's starting persona and completed evidence.
2. Run the Level 3 exit rubric before assigning the capstone.
3. Give the same system brief and timebox to each learner.
4. Have two reviewers score independently before discussion.
5. Record completion time, missing prerequisites, score disagreement, and reviewer questions.
6. Update `pilot-calibration.md` with anonymized findings.
7. Change scope/rubric only with a written rationale and rerun the affected pilot if material.

## Exit Gate

The capstone may move from pilot to stable curriculum only after three pilots, reviewer disagreement is understood, hidden prerequisites are corrected, and the maintainers approve the calibration record as a significant curriculum milestone.
