# Identity, Networking, and Security

| Statement | Best Fit |
| --- | --- |
| Team wants stricter platform security opinion and curated ingress patterns | OpenShift |
| Team wants AWS-native task IAM and simpler non-Kubernetes operations | ECS with Fargate |
| Team wants Kubernetes API plus AWS cloud identity integration | EKS |
| Team wants Azure-native managed Kubernetes integration | AKS |

Identity and ingress shape workload security, deployment architecture, and runtime ownership. They are core platform concerns, not late-stage details.
