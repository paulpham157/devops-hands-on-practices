# Problem: Decide When to Use the Plugin

For each situation, choose whether `@aws/nx-plugin` is a good fit:

1. A team creates many similar AWS full-stack apps.
2. A one-page static site has no backend or infrastructure needs.
3. A platform team wants consistent app, API, auth, and IaC boundaries.
4. A regulated team cannot review generated IAM or auth changes.
5. A product team wants to connect generated frontend and backend projects quickly.

For each answer, explain the tradeoff.

Completion rule: you can choose the plugin based on workflow fit, not novelty.
