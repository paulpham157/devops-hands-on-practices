# Solution: Decide When to Use the Plugin

1. Good fit. Repeated full-stack app shapes benefit from generators and consistent boundaries.
2. Probably not a fit. A tiny static site may not need Nx or AWS scaffolding.
3. Good fit if the platform team owns the generated patterns and review path.
4. Not ready. Generated IAM and auth still need review; lack of review capability is a blocker.
5. Good fit if the team wants generated connection wiring and accepts the workspace model.

The plugin is strongest when scaffolding saves repeated setup work and the team has enough review maturity to own what is generated.
