# Tutoring Patterns

Use these patterns to keep the learner engaged without over-solving the lesson.

## 1. Lesson Kickoff Template

Use when the learner starts a new lesson.

1. State the lesson goal in plain language.
2. Name 2 to 4 prerequisites.
3. Give a short pre-lab checklist.
4. Tell the learner what success looks like.

Suggested shape:

- Goal
- What to know first
- What to run first
- What you should be able to explain at the end

## 2. Hint Ladder

Use this when the learner is stuck on an exercise.

### Level 1: Nudge

Restate the problem in simpler words and point to the exact file or command to inspect.

### Level 2: Concrete hint

Name the relevant concept and one likely place the learner should change or verify.

### Level 3: Verification hint

Tell the learner exactly what command or output would confirm the fix.

### Level 4: Partial answer

Provide the critical fragment, structure, or reasoning step without giving the whole solution.

### Level 5: Full answer

Give the worked answer only after the learner has attempted the task or explicitly asks for it.

## 3. Debugging Template

Use when the learner pastes command output or validation failures.

1. Quote the failing command or error.
2. Translate the error into plain language.
3. Explain the lesson concept behind the failure.
4. Give the next smallest corrective action.
5. Say what to run next to verify.

## 4. End-of-Lesson Assessment

End lessons with:

- 3 factual recall questions
- 1 reasoning question
- 1 production tradeoff question

Example tradeoff prompts:

- Why would a team choose this pattern instead of a simpler one?
- What risk does this command or manifest reduce?
- What would need to change for production?
