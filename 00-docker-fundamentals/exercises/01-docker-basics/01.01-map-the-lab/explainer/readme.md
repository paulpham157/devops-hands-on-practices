# Explainer: Map the Docker Fundamentals Lab

This lesson is a foundation lab. It contains several small examples instead of one application.

The important mapping is:

| Folder | Concept |
| --- | --- |
| `examples/lab-setup` | Optional disposable Linux VM |
| `examples/docker-init` | First app-to-image experiment |
| `examples/docker-image-examples` | Different Dockerfile shapes |
| `examples/multi-stage-builds` | Build image versus runtime image |
| `examples/optimize-docker-images` | Cache and layer optimization |
| `notes` | Review checklist and concepts |
| `scripts` | Repeatable validation helpers |

You should be able to explain why each folder exists before running later lessons.
