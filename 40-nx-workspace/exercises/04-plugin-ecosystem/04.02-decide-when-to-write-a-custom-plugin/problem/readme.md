# Problem: Decide When To Write A Custom Plugin

Open:

```text
plugin-catalog/custom-plugin-notes.md
```

For each case, choose standard plugin, custom plugin, or no plugin:

1. A team repeats the same service scaffold 20 times per quarter.
2. A repo has one unusual shell script run by one team.
3. An internal API schema file should create project graph dependencies.
4. A standard Nx plugin already supports the test tool.
5. A platform team needs migrations for generated service templates.

Completion rule: you can justify custom plugin maintenance cost.
