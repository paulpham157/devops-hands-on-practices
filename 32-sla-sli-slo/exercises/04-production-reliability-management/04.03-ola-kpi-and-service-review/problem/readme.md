# OLA, KPI, and Service Review

For the reference order flow, write a short service review containing:

1. One user-facing SLI and SLO.
2. One customer-facing SLA consequence, if appropriate.
3. One OLA between the application team and Platform team, including response and escalation expectations.
4. Two KPIs: one delivery or operational KPI and one cost or adoption KPI.
5. The decision each measure should trigger when it worsens.

Keep the OLA internal, keep the SLA externally meaningful, and explain why each KPI is not itself an SLO.
