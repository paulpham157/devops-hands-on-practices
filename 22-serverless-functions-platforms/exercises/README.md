# Exercises: Serverless Functions Platforms

Work through these exercises after reading the lesson README.

## Exercise Map

1. `01-serverless-fundamentals/01.01-understand-serverless-model`
   - Understand what the platform owns and what the developer still owns.

2. `01-serverless-fundamentals/01.02-compare-invocation-models`
   - Compare HTTP, event, schedule, edge, and background functions.

3. `02-platform-comparison/02.01-map-six-platforms`
   - Map AWS Lambda, Cloudflare Workers, Azure Functions, Vercel, Netlify, and GCP Functions.

4. `02-platform-comparison/02.02-choose-platform-for-use-case`
   - Choose a platform for realistic serverless use cases.

5. `03-function-design/03.01-write-portable-handler-shapes`
   - Read handler shapes across six platforms.

6. `03-function-design/03.02-events-bindings-and-integrations`
   - Design event triggers, bindings, queues, and storage integrations.

7. `04-production-readiness/04.01-cold-starts-cost-and-limits`
   - Reason about cold starts, duration, memory, concurrency, and cost.

8. `04-production-readiness/04.02-security-observability-and-deployments`
   - Apply production security, observability, and deployment practices.
