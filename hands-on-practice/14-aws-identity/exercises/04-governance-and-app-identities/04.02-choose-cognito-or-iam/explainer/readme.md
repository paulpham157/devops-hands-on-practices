# Explainer: Choose Cognito or IAM

IAM is for access to AWS APIs and AWS resources.

Cognito is for application user identity, such as customers signing into a web or mobile app.

Use IAM for:

- Operators
- AWS workloads
- Automation
- Cross-account access
- AWS service permissions

Use Cognito or another app identity provider for:

- Customer sign-up and sign-in
- App sessions
- User pools
- Social identity federation
- App-facing tokens
