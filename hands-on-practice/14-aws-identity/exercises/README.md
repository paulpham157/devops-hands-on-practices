# Exercises: AWS Identity Types

## Sections

1. `01-identity-basics`
   - Classify AWS identity types.
   - Choose human vs workload identity.

2. `02-iam-policy-model`
   - Read an identity policy.
   - Read a trust policy.

3. `03-temporary-credentials-and-federation`
   - Map STS AssumeRole.
   - Compare IAM Identity Center and IAM users.

4. `04-governance-and-app-identities`
   - Understand SCP guardrails.
   - Choose Cognito or IAM for app identity.

## Completion Target

By the end, you should be able to explain:

- Why roles and temporary credentials are usually safer than static access keys.
- Why a role needs both a trust policy and permission policy.
- Why IAM Identity Center is preferred for workforce access.
- How SCPs limit maximum permission across accounts.
- When Cognito is the right identity service and when IAM is the wrong abstraction.
