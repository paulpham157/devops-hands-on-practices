# Solution: Choose Runtime for a Service

Reasonable answers:

1. Python or Go. Python is productive for automation; Go is good for a small deployable service.
2. Java Spring. Spring has strong enterprise patterns and integration support.
3. Go or Rust. Both can produce small runtime images and low-overhead services.
4. Ruby or Python. Both are productive for small web teams.
5. Rust. It gives strong memory safety and low runtime overhead.
6. Go or Rust. Both produce portable binaries that fit CI helper tools well.

There is no single correct answer without team and platform context. The important part is to connect the runtime choice to operational constraints.
