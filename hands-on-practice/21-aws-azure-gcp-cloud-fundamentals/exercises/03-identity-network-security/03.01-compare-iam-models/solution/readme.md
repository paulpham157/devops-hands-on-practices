# Solution: Compare IAM Models

An AWS IAM role grants temporary permissions that can be assumed by users, services, or workloads.

An Azure managed identity lets Azure resources authenticate without storing credentials in code.

A Google Cloud service account represents a workload identity for applications and services.

Long-lived access keys are risky because they can leak, persist, and be reused until revoked.

Least privilege means granting only the permissions needed for a task, for the shortest reasonable scope and time.
