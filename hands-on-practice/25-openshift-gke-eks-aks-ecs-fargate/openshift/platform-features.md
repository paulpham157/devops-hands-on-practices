# OpenShift Platform Features

OpenShift is a Kubernetes platform with additional operational opinion.

## Common Platform Signals

- `Route` objects provide an OpenShift-native way to expose services externally.
- Security Context Constraints enforce pod security rules at the platform layer.
- Operators are a first-class operational model for extending cluster functionality.
- Platform teams often use OpenShift to standardize ingress, security posture, and lifecycle operations across many application teams.

## Practical Tradeoff

OpenShift reduces assembly work for enterprises that want a more curated platform, but it also means the team must learn OpenShift-specific features in addition to core Kubernetes.
