# Observability Boundaries and Ownership

1. Routing decisions, new-service health, rollback triggers, and remaining legacy dependency paths.
2. Translation failures, schema mismatches, rejected payloads, and external contract drift.
3. The app may appear healthy while the helper behavior such as logging, proxying, or secret sync is failing.
4. Unowned boundaries become stale, hard to debug, and risky to change.
