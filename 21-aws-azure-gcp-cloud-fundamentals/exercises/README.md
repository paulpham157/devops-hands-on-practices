# Exercises: AWS, Azure, and Google Cloud Fundamentals

Work through these exercises after reading the lesson README.

## Exercise Map

1. `01-cloud-models/01.01-compare-cloud-providers`
   - Compare AWS, Azure, and Google Cloud at a high level.

2. `01-cloud-models/01.02-regions-zones-resource-hierarchy`
   - Understand regions, zones, accounts, subscriptions, projects, and resource groups.

3. `02-core-services/02.01-map-compute-storage-networking`
   - Map compute, storage, networking, databases, containers, and observability services.

4. `02-core-services/02.02-choose-managed-services`
   - Decide when to use managed services versus self-managed services.

5. `03-identity-network-security/03.01-compare-iam-models`
   - Compare AWS IAM, Azure RBAC/Microsoft Entra ID, and Google Cloud IAM.

6. `03-identity-network-security/03.02-design-secure-network-baseline`
   - Design a safe baseline network for a cloud workload.

7. `04-operations-costs/04.01-cli-and-tagging-practice`
   - Read provider CLI examples and define a tagging or labeling standard.

8. `04-operations-costs/04.02-cost-reliability-and-migration-tradeoffs`
   - Reason about cost controls, reliability targets, and provider migration tradeoffs.
