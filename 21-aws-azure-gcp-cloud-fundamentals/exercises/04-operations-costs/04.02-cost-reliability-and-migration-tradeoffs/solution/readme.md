# Solution: Cost, Reliability, and Migration Tradeoffs

Map first:

- container platform
- managed PostgreSQL
- object storage
- secret manager
- logging and monitoring
- load balancer
- network design

Collect:

- current monthly spend by service
- data transfer costs
- storage growth
- database size and IOPS
- support costs
- operational labor

Preserve:

- uptime target
- backup and restore targets
- latency requirements
- security controls
- audit logging

Hidden costs:

- migration engineering
- retraining
- networking changes
- data transfer
- new monitoring and security integration
- rewritten automation

Staying may be better when savings are uncertain, operational risk is high, or the current platform already meets requirements.
